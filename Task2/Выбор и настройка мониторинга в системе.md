### Мотивация

Система мониторинга и алёртинга позволит компании «Александрит» повысить надёжность обработки заказов и сократить количество инцидентов, напрямую
влияющих на бизнес.

На текущий момент компания сталкивается с потерей заказов, задержками в обработке и жалобами клиентов (как B2C, так и API-партнёров). Отсутствие
прозрачности в работе распределённой системы (Shop → CRM → MES через RabbitMQ) не позволяет быстро выявлять причины проблем и устранять их.

Внедрение мониторинга даст следующие преимущества:

* Снижение потерь заказов:
  Позволит отслеживать сбои в очередях и интеграциях (например, зависшие или потерянные сообщения в RabbitMQ).
* Ускорение реакции на инциденты:
  Алёртинг позволит оперативно реагировать на рост ошибок (HTTP 500), деградацию производительности и перегрузку сервисов.
* Повышение SLA для API-клиентов:
  Мониторинг latency и ошибок позволит контролировать качество API и удерживать партнёров.
* Оптимизация производительности MES и CRM:
  Метрики помогут выявить узкие места (например, медленные запросы или перегрузку CPU), влияющие на работу операторов.
* Прозрачность жизненного цикла заказа:
  Наблюдаемость (observability) позволит понимать, на каком этапе теряются или задерживаются заказы.
* Обоснование масштабирования:
  Метрики нагрузки (RPS, CPU, memory) позволят принимать решения о масштабировании инфраструктуры на основе данных, а не предположений.

Таким образом, мониторинг напрямую влияет на выручку компании, снижая количество потерянных заказов и повышая удовлетворённость клиентов.

## Выбор подхода к мониторингу

### API (MES, Shop, CRM): 4 золотых сигнала.

#### Latency:

* p95 / p99 response time (excluding background processing)

#### Traffic:

* Number of requests (RPS)
* Number of requests (RPS) per user (for detecting abusive clients)

#### Errors:

* Number of HTTP 5xx
* Error rate (5xx / total requests)
* Number of HTTP 4xx (client errors)

#### Saturation:

* CPU %
* Thread pool utilization
* Memory utilization

#### DBs (Shop, MES) — USE

Utilization:

* Memory utilization
* Number of connections

Saturation:

* Slow queries
* Lock wait / active locks
* Connection saturation (% of used connections from max pool)
* CPU utilization
* Disk I/O latency

Errors:

* Query errors
* Connection errors / timeouts

#### 3D file storage

Capacity:

* Size of S3 storage

Saturation:

* Disk I/O latency

#### RabbitMQ

* Number of messages in queue (queue length)
* Number of messages in flight
* Message processing lag (time in queue)
* Messages published per second
* Messages consumed per second
* Number of dead-letter-exchange messages
* Acknowledgement rate
* Retry rate
* CPU utilization
* Memory utilization
* Disk usage (for durable queues)

#### Business metrics

* Number of orders per status
* Time between status transitions
* Number of stuck orders (no status change > N minutes)
